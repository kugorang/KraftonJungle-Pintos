Advanced Scheduler

고급 스케줄러(Advanced Scheduler) 구현은 Pintos의 멀티레벨 피드백 큐 스케줄러를 4.4BSD 스케줄러와 유사하게 만드는 것을 목표로 합니다. 이 스케줄러는 시스템에서 실행 중인 작업의 평균 응답 시간을 줄이는 데 도움이 됩니다.

고급 스케줄러의 특징

  • 우선순위 기반 실행: 고급 스케줄러 역시 우선순위를 기반으로 스레드를 실행합니다. 하지만 우선순위 기부(priority donation)는 수행하지 않습니다.
  • 스케줄링 알고리즘 선택: Pintos 시작 시 스케줄링 알고리즘 정책을 선택할 수 있어야 합니다. 기본적으로 우선순위 스케줄러가 활성화되어 있으며, -mlfqs 커널 옵션을 통해 4.4BSD 스케줄러를 선택할 수 있습니다.

4.4BSD 스케줄러

  • 다양한 스케줄링 요구 사항의 균형: 일반적인 목적의 스케줄러는 스레드의 다양한 스케줄링 요구 사항을 균형있게 조정합니다. I/O가 많은 스레드는 빠른 응답 시간이 필요하며, 계산 집약적인 스레드는 많은 CPU 시간이 필요합니다.
  • 멀티레벨 피드백 큐: 여러 개의 준비 큐를 유지하며, 각 큐는 다른 우선순위의 스레드를 포함합니다. 가장 높은 우선순위의 비어 있지 않은 큐에서 스레드를 선택합니다.
  • 동적 우선순위 결정: 스레드의 우선순위는 스케줄러에 의해 동적으로 결정되며, 각 스레드는 다른 스레드에 얼마나 "친절하게" 행동할지 결정하는 nice 값도 가집니다.

Niceness

  • 우선순위 계산: 우선순위는 PRI_MAX - (recent_cpu / 4) - (nice * 2) 공식을 통해 계산되며, recent_cpu는 최근 CPU 사용량의 추정치이고, nice는 스레드의 nice 값입니다.

recent_cpu 및 load_avg 계산

  • recent_cpu: 스레드가 최근에 사용한 CPU 시간의 추정치로, 타이머 인터럽트가 발생할 때마다 실행 중인 스레드의 recent_cpu가 1씩 증가합니다.
  • load_avg: 시스템의 평균 부하를 추정하며, 실행 중이거나 실행 준비가 된 스레드의 수를 기반으로 계산됩니다.

구현해야 할 함수

  • int thread_get_nice(void): 현재 스레드의 nice 값을 반환합니다.
  • void thread_set_nice(int nice): 현재 스레드의 nice 값을 설정하고 우선순위를 재계산합니다.
  • int thread_get_recent_cpu(void): 현재 스레드의 recent_cpu 값을 반환합니다.
  • int thread_get_load_avg(void): 현재 시스템의 load_avg 값을 반환합니다. 이 값은 시스템의 전체 부하 평균을 나타내며, 실행 중이거나 준비 상태에 있는 스레드 수의 평균을 기반으로 계산됩니다.

고급 스케줄러 구현 시 고려사항

  • 스케줄러 선택: Pintos 시작 시 -mlfqs 옵션을 통해 스케줄러를 선택할 수 있어야 하며, 이는 thread_mlfqs 변수를 통해 관리됩니다.
  • 우선순위 계산 무시: 4.4BSD 스케줄러가 활성화되면, 스레드는 더 이상 자신의 우선순위를 직접 제어할 수 없습니다. thread_create()의 우선순위 인자와 thread_set_priority(), thread_get_priority() 호출은 무시됩니다.

Fixed-Point Real Arithmetic 구현

  • 고급 스케줄러에서는 실수 연산이 필요하지만, Pintos 커널은 부동소수점 연산을 지원하지 않습니다. 따라서, 고정소수점 산술을 사용하여 실수 연산을 시뮬레이션해야 합니다.
  • 이는 정수의 오른쪽 비트를 소수부로 취급하여 실수를 표현하는 방식입니다. 예를 들어, 17.14 고정소수점 표현에서는 32비트 정수의 하위 14비트를 소수부로 사용합니다.
  • 변환: 정수를 고정소수점으로 변환하거나, 고정소수점을 정수로 변환할 때는 적절한 승수(f = 2^q)를 사용합니다.
  • 연산: 고정소수점 수끼리의 덧셈, 뺄셈, 곱셈, 나눗셈 연산을 수행할 때는 정의에 따라 적절한 조정을 해야 합니다.
