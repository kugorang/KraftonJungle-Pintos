FAQ

FAQ

이 FAQ 섹션은 Pintos 프로젝트를 진행하면서 발생할 수 있는 다양한 질문들에 대한 답변을 제공합니다. 예를 들어 코드 작성량, Makefile 업데이트 방법, 컴파일러 경고 메시지, 타이머 인터럽트 간격, 타임 슬라이스 길이, 테스트 실행 방법 등에 대한 정보를 제공합니다.

코드 양

  • 참조 솔루션에 기반한 git diff --stat 요약은 6개 파일에서 총 330줄이 추가되고 12줄이 삭제된 것을 보여줍니다.
  • 이는 가능한 솔루션 중 하나에 불과하며, 여러 다른 솔루션도 가능하며 크게 다를 수 있습니다.

Makefile 업데이트 (새 소스 파일 추가 방법)

  • 새로운 .c 파일을 추가할 때는 해당 서브디렉토리의 targets.mk 파일을 편집하고, 새 파일을 dir_SRC 변수에 추가합니다.
  • 새 .h 파일을 추가할 때는 Makefiles를 편집할 필요가 없습니다.

컴파일러 경고 메시지

  • "no previous prototype for func" 경고는 비정적 함수를 프로토타입 없이 정의했을 때 발생합니다.
  • 해결을 위해 해당 함수를 헤더 파일에 프로토타입으로 추가하거나, 다른 .c 파일에서 사용되지 않는 경우 함수를 static으로 만듭니다.

타이머 인터럽트 간격

  • 타이머 인터럽트는 TIMER_FREQ로 정의된 횟수만큼 초당 발생합니다.
  • 기본값은 100Hz이며, 이 값을 변경하는 것은 권장되지 않습니다.

타임 슬라이스 길이

  • 타임 슬라이스는 TIME_SLICE 매크로로 정의되며, 기본값은 4 틱입니다.
  • 이 값 역시 변경을 권장하지 않습니다.

테스트 실행 방법

  • make check 명령을 사용하여 관련 테스트를 실행할 수 있습니다.

기타 FAQ

우선순위 스케줄링과 기아 현상

  • 엄격한 우선순위 스케줄링은 더 높은 우선순위를 가진 스레드가 실행 가능한 경우 낮은 우선순위의 스레드가 실행되지 않게 할 수 있습니다.
  • 고급 스케줄러는 스레드 우선순위를 동적으로 변경하는 메커니즘을 도입하여 이 문제를 해결합니다.

락 해제 후 실행할 스레드

  • 락이 해제될 때, 해당 락을 기다리는 가장 높은 우선순위의 스레드가 차단 해제되어 준비 리스트에 추가되어야 합니다.
  • 스케줄러는 준비 리스트에서 가장 높은 우선순위의 스레드를 실행해야 합니다.

이 FAQ는 Pintos 프로젝트를 진행하는 데 있어 핵심적인 정보를 제공하며, 프로젝트의 성공적인 수행을 위한 지침과 팁을 제공합니다. 프로젝트를 진행하면서 발생할 수 있는 일반적인 질문에 대한 답변을 찾는 데 유용합니다.
